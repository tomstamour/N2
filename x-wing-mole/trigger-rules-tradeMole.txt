================================================================================
trigger-rules-tradeMole.txt
--------------------------------------------------------------------------------
Notes only. No effect on the code. Documents every rule that gates the
trade-mole buy-signal (the surge/trigger).

All rules live in ONE place in trade-mole-2.0.py:
    IBKRSurgeApp.should_fire(w, iti)   (the swappable composite trigger rule)
The buy-signal fires only when ALL conditions below are TRUE at the same trade
event. When it fires, the trigger row is recorded, recording stops
(stop_at_trigger), and buy_signal_callback(last_ask) is handed to x-wing.
================================================================================


TRIGGER FIRES IFF (logical AND of every condition):

1. MIN TRADES IN 5s
   condition : trades_in_5s >= surge_min_trades_5s
   CLI       : --surge-min-trades-5s
   default   : 5
   purpose   : absolute activity floor; ignore thin/illiquid bursts.

2. DOLLAR-VOLUME RATE (5s)
   condition : dollar_rate_5s >= surge_dollar_rate     (USD/sec over the 5s window)
   CLI       : --surge-dollar-rate
   default   : 300.0
   purpose   : require real money flow, not just trade count. Strongest
               separator observed between the positive and negative cases.

3. SPREAD TIGHTNESS (current)
   condition : current spread_pct <= surge_max_spread_pct
               spread_pct = (ask - bid) / midprice
   CLI       : --surge-max-spread-pct
   default   : 0.02  (2%)
   purpose   : avoid entering when the book is wide/illiquid.

4. BUYER-INITIATED RATIO (5s)
   condition : lift_offer_ratio_5s >= surge_min_lift_ratio
               lift_offer_ratio = buy_count / (buy_count + sell_count)
               (aggressor classified vs prevailing NBBO at trade time)
   CLI       : --surge-min-lift-ratio
   default   : 0.55  (>=55% buyer-initiated)
   purpose   : confirm the burst is buyers lifting the offer, not sellers.

5. BID DRIFT (5s) — short-horizon non-falling guard
   condition : bid_drift_5s_bp >= surge_min_bid_drift_bp
               bid_drift_5s_bp = (bid_now - bid_5s_ago) / bid_5s_ago * 10000
   CLI       : --surge-min-bid-drift-bp
   default   : 0.0  (bid must be non-falling over the last 5s)
   purpose   : block entries while the bid is sliding down in the last 5s.

6. NO NET SELLOFF SINCE START — long-horizon non-falling guard   [added]
   condition : (last_price - start_price) / start_price >= -surge_max_price_drop_pct
               start_price = price of the FIRST trade of the session (anchor)
               last_price  = most recent executed trade price
   CLI       : --surge-max-price-drop-pct
   default   : 0.005  (block if last price is >0.5% below the start price)
   purpose   : do NOT enter on a stock that is in a net selloff since the
               trade-mole started watching, even if a short buyer-burst occurs.
   notes     : complements rule 5 (which only looks back 5s; the trade deque is
               trimmed to 10s). This one spans the whole session.
               0   = block ANY net drop (strict).
               1.0 = effectively disables this guard (restores prior behavior).
   column    : price_change_since_start_pct is recorded on every TRADE row for
               post-hoc analysis.
